Conducting OS fingerprinting with xprobe2

We can use nmap or hping2 to do Operating System Fingerprinting, but they simply aren't as reliable or accurate as xping3, a tool specifically built for the purpose of OS fingerprinting.

xprobe2 is one of the best active tool for active OS fingerprinting, which means that xprobe2 probes the target system and determines the Operating System from the system's response. Xprobe2 has a total of 16 different modules it runs in order to help determine the OS.

Any fingerprinting that probes a network with special packets is going to be really noisy and is likely to be detected by the Network Intrusion Detection System and othe security system.

The syntax for running an xprobe fingerprint is:

  • xprobe2 <target domain or IP>

If you run nmap -O you should get similar results, thought they tend to be less accurate and reliable.

As we mentioned earlier, xprobe2 has 16 modules which it uses in order to determine the OS of the target, and determines the probability of each guess. We can look at these modules:

  • xprobe2 -L

Xprobe2 uses all the 16 modules by default, but we can remove modules we don't need when probing a system using the -D switch. For example, on many network, the icmp modules will hang and it may be neccessary to remove these modules in order to get xprobe2 to work. We can do this:

  • xprobe2 -D google.com
